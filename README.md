# ОКНА CRM — демо для оконного/стекольного бизнеса

Демо-CRM для производства светопрозрачных конструкций (окна, стекло, фасады).
Воронка продаж, замер с расчётом и КП, склад с приходом/списанием по этапам,
канбан производства, дебиторка. Чистый статический сайт — **vanilla HTML/CSS/JS, без сборки**.

🔗 Прод: https://pllato.kz/app/okna/

## Структура

```
index.html      — точка входа (монтирует #app)
styles.css      — стили
data.js         — демо-данные (моки сделок, клиентов, склада)
ui.js           — UI-хелперы (рендер, модалки, тосты)
modules.js      — модули: воронка, замер, КП …
modules2.js     — модули: склад, производство, дебиторка …
main.js         — роутер/инициализация приложения
site/           — лендинг клиента Ocean Glass (v1/v2/v3) + КП/счёт
```

Скрипты подключаются по порядку: `data.js → ui.js → modules.js → modules2.js → main.js`.

## Запуск локально

Сборка не нужна. Любой статический сервер из корня репозитория:

```bash
python3 -m http.server 8080
# открыть http://localhost:8080/
```

Или просто открыть `index.html` в браузере.

## Деплой

Хостинг статический — **Cloudflare Pages** (git-интеграция):

| Настройка | Значение |
|---|---|
| Framework preset | None |
| Build command | *(пусто)* |
| Build output directory | `/` |
| Root directory | `/` |

Каждый merge в `main` → автодеплой. Превью-деплой создаётся для каждого Pull Request.

## Как работаем (для команды)

1. Ветка от `main`: `git checkout -b feature/<что-делаем>`
2. Коммит, push, **Pull Request** в `main`.
3. CI/Pages соберёт превью PR — проверяем.
4. Ревью → merge в `main` → автодеплой в прод.

Не коммитьте секреты/ключи. Демо-данные — в `data.js`.
