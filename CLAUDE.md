# CLAUDE.md

Гайд для работы с этим репозиторием. Демо-CRM для оконного/стекольного
производства (окна, стекло, фасады) — показывает клиентам Pllato полный цикл
сделки на вымышленных данных.

🔗 Прод: https://pllato.kz/app/okna/

## Стек и принципы

- **Чистый vanilla HTML/CSS/JS. Без сборки, без npm, без зависимостей.**
  Единственный внешний ресурс — шрифт Inter с Google Fonts.
- Рендер — строковые шаблонные литералы (`` ` ``) → `innerHTML`. Нет
  виртуального DOM: меняешь `state` и вызываешь `render()` (полная
  перерисовка) или `renderModule()` (перерисовка только текущего экрана).
- Данные хранятся в **localStorage** (ключ `okna_crm_db_v1`), при первом
  заходе сидятся моками из `buildSeed()`. После любой мутации зови `saveDB()`.
- Взаимодействие — **делегирование событий** на `document` через атрибуты
  `data-act` / `data-id` / `data-*`. Весь роутинг кликов — один `switch`
  в `main.js`. Не вешай инлайновые `onclick`.

## Структура файлов

```
index.html      точка входа: #app, #modal-root, #toast-root; подключает скрипты
styles.css      стили; тёмная/светлая тема через [data-theme] + CSS-переменные
data.js         модель: иконки, хелперы, статкаталог, buildSeed (моки),
                load/save DB, демо-гейт по ссылке, права доступа, pricing
ui.js           каркас: render(), gate, login, shell (сайдбар/топбар),
                openModal/closeModal, toast(), UI-хелперы (kpi, bars, аватары)
modules.js      экраны: Дашборд, Воронка (+модалка сделки), Клиенты
modules2.js     экраны: Замер+КП, Склад, Производство, Финансы, Настройки
main.js         actions (мутации), event-делегирование, drag&drop, init render()
site/           отдельный лендинг клиента Ocean Glass (v1/v2/v3) + КП/счёт
```

**Порядок загрузки обязателен** (каждый файл опирается на глобали предыдущего):
`data.js → ui.js → modules.js → modules2.js → main.js`. При добавлении файла
вставь `<script>` в `index.html` в правильное место.

## Поток данных

- Глобальный `DB` (из localStorage) — все сущности: `company`, `users`,
  `materials`, `components`, `clients`, `deals`, `payables`, `activity`.
- Глобальный `state` — UI-состояние: `user`, `module`, `measureDealId`,
  `financeTab`, `whTab`, `sideOpen`, `theme`.
- Lookups: `userById`, `clientById`, `dealById`, `matById`, `compById`,
  `glassById`, `openById`.
- `render()` решает: гейт → логин → shell+модуль. `renderModule()` выбирает
  `render<Экран>()` по `state.module`, проверив `canSee()`.

## Экраны

| Модуль | Файл | Суть |

| Дашборд | modules.js | KPI, воронка, выручка по месяцам, менеджеры, источники, лента |
| Воронка | modules.js | Канбан сделок по стадиям, drag&drop, модалка сделки |

| Замер и КП ⭐ | modules2.js | Калькулятор конструкций, живой расчёт, генерация КП, аванс |
| Склад | modules2.js | Остатки профиля/комплектующих, «ниже минимума», приход |
| Производство | modules2.js | Канбан по этапам цеха, списание со склада по этапам |

Замер и КП — главный «вау»-экран: замерщик собирает заказ, система сразу
считает и формирует КП. Стадии сделки — `STAGES`, этапы цеха — `PROD_STAGES`.

## Ключевые механики

- **Права доступа** (`MODULE_ROLES` в `data.js`): роли director, manager,
  surveyor, production, warehouse. `canSee(mod)` гейтит модули, `seesMoney()`
  прячет деньги от сборщиков/склада. Логин = выбор роли. Меняешь видимость
  модуля — правь `MODULE_ROLES`.
- **Ценообразование** (`data.js`): `constrPrice` — площадь×(профиль+стекло) +
  открывание×створки + опции. `dealItemsSum`, `dealPaid`, `dealDebt`.
- **Производство ↔ склад**: `consumeForStage` списывает материалы при переходе
  по этапам (профиль на резке, стеклопакеты на «стеклопакет», фурнитура и
  опции на сборке), отмечает флаги в `d.consumed`.
- **Демо-гейт по ссылке** (`data.js`): подписанные токены с TTL (`?demo=…`),
  разблокировка владельца (`?owner=…`), хранится в localStorage. Подпись НЕ
  криптостойкая — задача мягко ограничить доступ, не защищать секреты.
- **Темы**: light/dark, переключатель в топбаре, в localStorage (`okna_crm_theme`).

## Конвенции

- Язык интерфейса и комментариев — русский. Валюта — тенге (`money()`,
  `moneyK()`). Даты через хелперы (`dateStr`, `dateFull`), «сейчас» — `SEED_NOW`.
- Иконки — инлайновые SVG-пути в объекте `ICON`, выводятся `icon(name, cls)`.
- Новый action: добавь функцию-мутацию и кейс в `switch` в `main.js`; в разметке
  повесь `data-act="..."`. После мутации DB — `saveDB()` и пере-рендер.
- Не коммить секреты/ключи. Демо-данные — только в `data.js`.

## Запуск и деплой

Сборки нет. Любой статический сервер из корня:

```bash
python3 -m http.server 8080   # → http://localhost:8080/
```

Хостинг — **Cloudflare Pages** (git-интеграция, preset None, build command
пустой, output `/`). Merge в `main` → автодеплой в прод; для каждого PR —
превью-деплой.

## Процесс

**Рабочий режим (по договорённости с владельцем):** коммитим **напрямую в
`main`** — без отдельной ветки, без PR, без ручного подтверждения. После
каждого изменения: правка → `commit` → `push origin main`. Merge в `main`
триггерит автодеплой Cloudflare на прод.

Этот режим выбран сознательно для быстрой выкатки демо. Если нужен ревью/превью
перед продом — заводи ветку и PR вручную по явной просьбе.
