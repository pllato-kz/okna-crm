# WhatsApp (Green API) — подключение

Код в CRM готов: входящие → создаётся клиент+сделка (источник WhatsApp), исходящие
отправляются из карточки. Нужно только привязать инстанс Green API с номером.

---

## Предусловие
- **Номер WhatsApp** для бизнеса (отдельная SIM/номер — не личный телефон сотрудника).
- **Green API инстанс**, к которому этот номер привязан (QR-сканирование в кабинете Green API).
- Из кабинета Green API: **idInstance** и **apiTokenInstance**.

## Шаг 1. Прописать инстанс в CRM
Войти директором → **Настройки → WhatsApp (Green API)**:
- вставить **idInstance** и **apiTokenInstance** → сохранить.
- (токен хранится на сервере в D1, в браузер не отдаётся).

## Шаг 2. Подключить приём входящих (вебхук)
Там же, кнопка **«Подключить приём»** — она зарегистрирует в Green API вебхук на адрес:
```
https://crm.oceanglass.kg/api/wa/webhook
```
(или текущий `https://okna-crm-1qh.pages.dev/api/wa/webhook`, пока домен не активен).

Если регистрируете вебхук вручную в кабинете Green API — укажите этот URL и тот же
webhook-секрет, что сохранён в настройках.

## Шаг 3. Проверка
- **Исходящее:** в карточке клиента → «Быстрое сообщение» / «Чат WhatsApp» → отправить.
- **Входящее:** написать на бизнес-номер с другого телефона →
  - новый номер → автоматически создаётся **клиент + сделка** (стадия «Новый лид», источник WhatsApp);
  - известный номер → сообщение падает в «Чат WhatsApp» этого клиента (без дублей).

## Заметки
- Тарифицируется Green API (платный инстанс) — см. `docs/BILLING.md`.
- Один новый номер = одна автосделка; повторные сообщения дублей не создают.
- Номер должен быть «прогрет» (не новый-новый), иначе WhatsApp может банить за рассылку.
