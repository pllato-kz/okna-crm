# ROLLOUT — пошаговая передача проекта клиенту «с нуля»

Полный порядок действий: от создания отдельной почты до боевой эксплуатации.
Идти строго по шагам. 👤 — делает клиент (или Pllato от его имени), 🤖 — Pllato.

> Идея: завести **единую «владельческую» личность** клиента (одна почта → на неё
> GitHub и Cloudflare). Тогда клиент реально владеет кодом и инфраструктурой,
> а Pllato помогает по доступу.

---

## Этап 0. Отдельная почта клиента
1. 👤 Создать **отдельный Gmail** под проект, напр. `oceanglass.crm@gmail.com`
   (или ящик на домене `oceanglass.kg` — что удобнее).
   - Это будет логин для **GitHub** и **Cloudflare**, и сюда же — все уведомления.
2. 👤 Стойкий пароль + **2FA**. Сохранить в менеджере паролей (Bitwarden/1Password).
3. 👤 (опционально) Завести рабочие ящики сотрудников для входа в CRM:
   `director@…`, `manager@…`, `zamer@…`, `sklad@…`, `ceh@…`.
   Если нет — на старте можно входить по личным email сотрудников.

---

## Этап 1. Перенос кода на GitHub клиента
4. 👤 Зарегистрировать **аккаунт GitHub** на почту из шага 1.
5. Перенести код одним из способов:
   - **Вариант A (сохранить историю) — передача репозитория:**
     🤖 в `github.com/pllato/okna-crm` → Settings → *Transfer ownership* → указать
     аккаунт клиента. 👤 принять перенос.
   - **Вариант B (чистый старт) — новый репозиторий:**
     👤 создать пустой приватный репо `okna-crm`.
     🤖 запушить актуальный код:
     ```bash
     git remote add client https://github.com/<client>/okna-crm.git
     git push client main
     ```
6. 👤 Выдать Pllato доступ к репозиторию (Collaborator), пока идёт настройка/поддержка.

---

## Этап 2. Аккаунт Cloudflare клиента
7. 👤 Зарегистрировать **Cloudflare** на ту же почту, подтвердить email, включить **2FA**.
8. 👤 Подключить **Pages-проект к GitHub-репозиторию** клиента:
   Cloudflare → Workers & Pages → Create → Pages → Connect to Git → выбрать `okna-crm`.
   - Build command — пусто; Output directory — `/` (preset None).
9. 👤 Пригласить Pllato членом аккаунта (Members → Invite) **или** выдать
   **API-токен** с правами Pages/D1/R2/Workers (Edit) + **Account ID** — чтобы 🤖 донастроил.

---

## Этап 3. База, файлы, секреты (🤖)
10. 🤖 Создать ресурсы на аккаунте клиента:
    ```bash
    wrangler d1 create okna-crm-db          # запомнить database_id
    wrangler r2 bucket create okna-crm-files
    ```
11. 🤖 Прописать новый `database_id` в `wrangler.toml`, закоммитить, запушить.
12. 🤖 Накатить схему и данные:
    ```bash
    wrangler d1 execute okna-crm-db --remote --file=schema.sql
    wrangler d1 execute okna-crm-db --remote --file=seed.sql     # справочники/стартовый каталог
    ```
13. 🤖 Задать секреты проекта (Pages → Settings → Variables and Secrets, либо CLI):
    - `JWT_SECRET` — **новый стойкий** (`openssl rand -hex 48`).
    - (позже) токены интеграций: WhatsApp, SIP.
14. 🤖 Дождаться авто-деплоя (push в main) либо `wrangler pages deploy .`. Проверить вход.

---

## Этап 4. Домен
15. 👤 Решить адрес: поддомен `crm.oceanglass.kg` (рекоменд.) или временно `*.pages.dev`.
16. 👤/🤖 В Pages → Custom domains добавить домен; DNS вести на Cloudflare (SSL автом.).

---

## Этап 5. Данные компании и каталог
17. 👤 Прислать реквизиты: юрлицо, город, адрес, телефон(ы), email.
18. 🤖 Внести в `Настройки → Данные компании`.
19. 👤 Прислать прайс/каталог: профили, стеклопакеты, опции, цены, остатки склада.
20. 🤖/👤 Внести каталог и начальные остатки (`Каталоги`, `Склад`).

---

## Этап 6. Учётные записи
21. 🤖 Переключить первого директора на реальные **имя + email + стойкий пароль**
    (вход существует только для уже заведённого аккаунта — первый создаётся в базе).
22. 👤 Директор входит и **сам заводит сотрудников**, выдаёт им пароли
    (`Настройки → Сотрудники`). Доступ к базе сотрудникам не нужен.

---

## Этап 7. Безопасность (финальная сверка)
23. 🤖 Убедиться: `JWT_SECRET` стойкий и задан; в `seed.sql` нет дефолт-пароля.
24. 👤 Все аккаунты — на стойких паролях (не `okna2026`).
25. ✅ Уже встроено: RBAC, ограничение попыток входа, security-заголовки, HTTPS,
    пароли PBKDF2, скрытие денег от сборщиков/склада.

---

## Этап 8. Интеграции (поэтапно, после старта)
26. 👤+🤖 **WhatsApp**: платный Green API инстанс + реальный номер, привязать вебхук.
27. 👤+🤖 **Instagram**: Business-аккаунт + Facebook-страница + сервис-форвардер;
    дать ссылку на инстаграм компании.
28. 👤+🤖 **Звонки (SIP)**: VM с Asterisk + SIP-trunk оператора + 2 секрета
    (`SIP_DOMAIN`, `SIP_ENDPOINT_PASSWORD`).

---

## Этап 9. Эксплуатация и приёмка
29. 🤖 Показать резервное копирование (экспорт/импорт) + регулярность.
30. 🤖 Обучение сотрудников по ролям (замер→КП, воронка, склад, цех, финансы).
31. 👤+🤖 Финальная приёмка: прогнать полный цикл сделки на боевых данных.
32. 👤+🤖 Передать доступы (почта, GitHub, Cloudflare), договориться о поддержке.

---

## Что нужно от клиента (сводка)
- Почта (шаг 1) → на неё GitHub + Cloudflare.
- Реквизиты компании + прайс/каталог.
- Имя/email реального директора.
- Решение по домену.
- Для интеграций (позже): WhatsApp-номер, Instagram Business + Facebook-страница.

## Критический путь до запуска
0 (почта) → 1 (GitHub) → 2 (Cloudflare) → 3 (база+JWT) → 4 (домен) →
5 (реквизиты+прайс) → 6 (директор+сотрудники) → 7 (пароли).
Интеграции — уже после старта.
